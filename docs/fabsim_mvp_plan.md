# FabSim MVP 计划书（单人2D版）

> 版本定位：最小可用产品，单人4-6周交付
> 技术栈：Godot 4.x + GDScript（全2D）

---

## 一、MVP 边界（明确不做什么）

| 不做 | 原因 |
|------|------|
| ❌ 3D渲染 | 2D拓扑图足够表达产线逻辑，开发成本降80% |
| ❌ 复杂设备模型 | 用标准色块+图标代替，后期再美化 |
| ❌ 多种调度算法 | MVP只实现FIFO，证明系统能跑通 |
| ❌ 实时MES联动 | 先支持CSV文件导入/导出，接口后期加 |
| ❌ 多人协作 | 单机运行，后续考虑联网 |

**MVP核心目标**：证明"仿真引擎+可视化+数据导出"这条链路能跑通。

### 模块3：仿真引擎（2周）

**核心：离散事件调度**

```gdscript
# 极简伪代码
class SimEngine:
    var event_queue = []      # 事件队列
    var current_time = 0      # 仿真时间（分钟）
    var entities = []         # 所有实体（Lot、Machine等）
    
    func run():
        while not queue.empty():
            var ev = queue.pop()
            current_time = ev.time
            handle_event(ev)
            
    func handle_event(ev):
        match ev.type:
            "LOT_ARRIVE":      # 新批次到达
                assign_to_buffer(ev.lot)
            "MACHINE_FREE":    # 设备空闲
                start_next_lot(ev.machine)
            "PROCESS_DONE":    # 加工完成
                move_to_next_step(ev.lot)
```

**功能**：
- 时间推进（支持1x/10x/100x加速）
- Lot状态流转：等待→加工→完成→下一工序
- 设备状态：空闲→加工→维护
- 基础FIFO调度

## 八、给你的建议 💡

1. **先让Lot动起来** — 哪怕设备都是方块，只要能看到Lot在流动，就有正反馈

2. **不要追求完美数据** — 先用硬编码的假数据跑通，再考虑导入导出

3. **一天一提交** — 小步快跑，每天有个能运行的版本

4. **先画再写** — 用纸笔画出界面布局，比空想省时间

5. **卡住就简化** — 如果一个功能想了一小时还没思路，先砍掉
