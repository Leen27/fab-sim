# 半导体工厂仿真系统 v0.1 设计文档

## 1. 系统定位

### 1.1 核心目标
本系统是 **MES 系统的效果验证与演示平台**，通过模拟半导体前道产线的运行，验证 MES 系统的核心功能：
- 工单派发与执行跟踪
- 设备状态监控与调度
- 工艺参数下发与采集
- 在制品 (WIP) 追溯
- 良率统计与异常预警

### 1.2 边界定义
| 在范围内 | 不在范围内 |
|---------|-----------|
| 与 MES 的接口交互仿真 | 物理化学过程仿真 |
| 设备状态机模拟 | 设备内部机械运动 |
| Bank（缓存区）流转逻辑 | AGV/天车物理搬运 |
| 工艺时间模拟 | 真实的工艺参数计算 |
| 物料流转逻辑 | 物料物理属性变化 |
| MES 指令响应验证 | 自动化设备控制 |
| **EAP功能仿真** | 真实SECS/GEM协议栈 |

### 1.3 使用场景
1. **MES 功能验证**：在部署到真实产线前，验证 MES 逻辑是否正确
2. **演示与培训**：向客户/新员工展示 MES 如何管理产线
3. **接口调试**：验证 MES 与设备层接口协议
4. **EAP功能测试**：模拟设备EAP行为，测试MES的EAP接口

### 2.3 Bank（缓存区）机制设计

#### 2.3.1 为什么需要 Bank？

在真实半导体工厂中，**Bank 是 MES 调度系统的核心枢纽**：

```
┌──────────────────────────────────────────────────────────────┐
│                     Bank 在产线中的位置                        │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│   上游设备          Bank (缓存区)           下游设备          │
│  ┌──────┐         ┌──────────┐           ┌──────┐           │
│  │ 沉积 │────────→│ 🏦       │──────────→│ 光刻 │           │
│  │ 完成 │ 释放    │ 批次队列  │  派工     │ 接收 │           │
│  └──────┘         └──────────┘           └──────┘           │
│                                                              │
│   MES 在这里做决策：                                          │
│   • 下游设备空闲？→ 立即派工                                  │
│   • 下游忙碌？   → 批次在 Bank 等待                           │
│   • 高优先级？   → 插队                                       │
│   • 设备维护？   → 换目标设备                                 │
└──────────────────────────────────────────────────────────────┘
```

**Bank 的核心作用：**
1. **解耦上下游**：上游设备完成后无需等待下游立即接收
2. **MES 调度点**：所有流转决策由 MES 在 Bank 节点下发
3. **WIP 暂存**：平衡产线节拍差异，防止堵塞
4. **追溯节点**：物料位置状态的关键上报点

#### 2.3.2 Bank 的类型

| Bank 类型 | 用途 | MES 管理内容 |
|---------|------|-------------|
| **入口 Bank** | 原料/初始晶圆投入 | 投料计划、批次校验 |
| **工序间 Bank** | 两道工序之间的缓存 | 等待队列、优先级排序 |
| **出口 Bank** | 成品/已完成批次 | 产出统计、质量放行 |

#### 2.3.3 Bank 的状态定义

```
┌─────────────────────────────────────────────────────────────┐
│                     Bank 状态流转图                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────┐    批次到达    ┌─────────┐                    │
│   │  EMPTY  │───────────────→│ HOLDING │                    │
│   │ (空闲)  │←───────────────│ (持有)  │                    │
│   └─────────┘   批次全部离开  └────┬────┘                    │
│                                    │                        │
│                   MES派工确认      ↓  等待目标设备            │
│                         ┌─────────────┐                     │
│                         │  RESERVED   │                     │
│                         │  (已预留)    │                     │
│                         └──────┬──────┘                     │
│                                │ 设备就绪/搬运完成           │
│                                ↓                            │
│                         ┌─────────────┐                     │
│                         │ TRANSFERRING│                     │
│                         │  (搬运中)    │                     │
│                         └──────┬──────┘                     │
│                                │ 批次离开                   │
│                                ↓                            │
│                         [回到 EMPTY 或 HOLDING]             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

| 状态 | 说明 | MES 上报时机 |
|-----|------|-------------|
| **EMPTY** | Bank 空闲，无批次 | - |
| **HOLDING** | 持有批次，等待 MES 派工 | 批次到达时上报 |
| **RESERVED** | MES 已确认派工，等待目标设备就绪 | 派工确认时上报 |
| **TRANSFERRING** | 批次正在移出 Bank | 搬运开始时上报 |

#### 2.3.4 Bank 的核心交互流程

```
┌──────────┐     工艺完成      ┌──────────┐     批次到达上报    ┌──────────┐
│  上游设备  │────────────────→│   Bank   │───────────────────→│   MES    │
│ (COMPLETE)│   Lot释放       │(HOLDING) │                   │          │
└──────────┘                 └────┬─────┘                   └────┬─────┘
                                  │                              │
                                  │  MES查询下游设备状态           │
                                  │←─────────────────────────────┘
                                  │                              │
                                  │  MES下发派工指令              │
                                  │←─────────────────────────────┘
                                  │
                                  ↓
┌──────────┐     搬运完成       ┌──────────┐     设备就绪上报    ┌──────────┐
│  下游设备  │←───────────────│   Bank   │───────────────────→│   MES    │
│  (SETUP)  │   Lot接收        │(EMPTY)   │                   │          │
└──────────┘                  └──────────┘                   └──────────┘
```

**关键上报事件：**
1. `BANK_LOT_ARRIVE` - 批次进入 Bank
2. `BANK_DISPATCH_REQUEST` - Bank 向 MES 请求派工
3. `BANK_DISPATCH_CONFIRM` - MES 确认派工
4. `BANK_LOT_DEPART` - 批次离开 Bank

#### 2.3.5 仿真系统中的 Bank 建模

在仿真系统中，Bank 是**逻辑节点**而非物理实体：

```
实际工厂                    仿真系统
┌──────────────┐           ┌──────────────┐
│   设备A      │           │   Equipment  │
│  ┌────────┐  │           │  ┌────────┐  │
│  │LoadPort│  │           │  │虚拟Load │  │ ← 简化为Bank对象
│  └────────┘  │           │  └────────┘  │
│      ↑↓      │    →     │      ↑↓      │
│  ┌────────┐  │           │  ┌────────┐  │
│  │ 设备本体 │  │           │  │Process │  │
│  └────────┘  │           │  └────────┘  │
└──────────────┘           └──────────────┘
```

**Bank 与设备的关联关系：**
- 每个设备可有 **输入 Bank**（上游来料缓存）
- 每个设备可有 **输出 Bank**（下游去向缓存）
- Bank 也可独立存在（如中央缓存区）

### 2.4 EAP (Equipment Automation Program) 仿真设计

#### 2.4.1 什么是 EAP？

EAP 是设备自动化程序，位于 MES 与设备之间，负责：
- 将 MES 的高级指令转换为设备可执行的 SECS 消息
- 将设备的 SECS 事件/数据上报给 MES
- 管理设备状态、配方 (PPID)、作业执行

```
┌─────────────────────────────────────────────────────────────┐
│                    EAP 在 CIM 架构中的位置                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│    MES 层 (SiView)                                          │
│    ┌─────────────────────────────────────────────────────┐ │
│    │  • 工单管理    • 设备调度    • WIP 追踪             │ │
│    └─────────────────────┬───────────────────────────────┘ │
│                          │ HTTP/REST API                    │
│                          ↓                                   │
│    EAP 层 ─────────────→┌─────────────────┐                │
│    (本仿真系统          │   EAP Adapter   │←── 模拟EAP功能  │
│     实现此层)           │                 │                │
│                         │ • S2F41 指令转换 │                │
│                         │ • S6F11 事件上报 │                │
│                         │ • S1F3/S1F4 数据采集│             │
│                         └────────┬────────┘                │
│                                  │ SECS-II (模拟)           │
│                                  ↓                          │
│    设备层                ┌─────────────────┐                │
│    (仿真设备)            │  Equipment FSM  │                │
│                          │                 │                │
│                          │ • Process State │                │
│                          │ • Control State │                │
│                          │ • Recipe执行    │                │
│                          └─────────────────┘                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 2.4.2 EAP 仿真核心目标

本仿真系统能够**模拟真实设备的 EAP 行为**，让 MES 以为在与真实设备通信：

| 能力 | 说明 | MES 验证点 |
|-----|------|-----------|
| **设备状态上报** | 模拟 S1F4 (设备状态查询响应) | MES 能否正确获取设备状态 |
| **事件上报** | 模拟 S6F11 (CEID 收集事件) | MES 能否接收并处理设备事件 |
| **远程指令** | 模拟 S2F41 (Host 命令) 接收与执行 | MES 能否远程控制设备 |
| **配方管理** | 模拟 S7F3/S7F5 (PPID 下载/上传) | MES 能否下发工艺配方 |
| **数据变量** | 模拟 S1F3/S1F4 (SV/EC 读写) | MES 能否采集过程数据 |

#### 2.4.4 EAP 仿真消息格式

本系统使用 **REST API 模拟 SECS-II 消息**，便于 MES 对接调试：

```json

  "stream_function": "S1F3",
  "equipment_id": "EQ001",
  "svids": [1, 2, 3, 4],
  "timestamp": "2024-01-15T08:30:00Z"
}

// 仿真系统 → MES (响应)
{
  "stream_function": "S1F4",
  "equipment_id": "EQ001",
  "status_variables": [
    {"svid": 1, "name": "ControlState", "value": "REMOTE"},
```

#### 2.4.5 设备状态与 GEM 状态映射

```
┌─────────────────────────────────────────────────────────────┐
│              设备状态机 ↔ GEM Control State 映射             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  仿真系统状态         GEM Control State                      │
│  ┌─────────┐                                         ┌───┐ │
│  │ OFFLINE │────────────────────────────────────────→│ 0 │ │
│  └────┬────┘                                         └───┘ │
│       │ 初始化完成                                        │
│       ↓                                                   │
│  ┌─────────┐     S1F17 (Online)      ┌─────────┐    ┌───┐ │
│  │ ONLINE  │────────────────────────→│ REMOTE  │───→│ 5 │ │
│  └────┬────┘                         └────┬────┘    └───┘ │
│       │                                  │                │
│       │ S2F41 (Local Request)            │ S2F41          │
│       │                                  │ (Remote)       │
│       ↓                                  ↓                │
│  ┌─────────┐     S1F17 (Online)      ┌─────────┐    ┌───┐ │
│  │  LOCAL  │←────────────────────────│ LOCAL   │───→│ 4 │ │
│  └─────────┘                         └─────────┘    └───┘ │
│                                                             │
│  Control State 说明:                                        │
│  • OFFLINE (0): 设备离线，无法通信                          │
│  • LOCAL (4): 本地控制模式，不接受Host指令                  │
│  • REMOTE (5): 远程控制模式，接受Host指令                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.5 AMHS 简化仿真设计

#### 2.5.1 为什么需要 AMHS 仿真？

在半导体工厂中，**AMHS（全自动物料搬运系统）** 是连接 Bank 与 Bank 的桥梁。MES 需要验证与 AMHS 的交互逻辑：

```
┌─────────────────────────────────────────────────────────────┐
│                    MES 与 AMHS 的交互                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   MES 决策层                                                 │
│   ┌─────────────────────────────────────────────────────┐  │
│   │ "LOT001在D1完成了，下一步去L1，发起搬运任务"           │  │
│   └────────────────────┬────────────────────────────────┘  │
│                        │ 发送搬运指令                        │
│                        ↓                                    │
│   AMHS 执行层          ┌──────────────┐                    │
│   ┌──────────────────→│  搬运任务队列  │                    │
│   │                   │  - LOT001     │                    │
│   │                   │  - LOT002     │                    │
│   │                   └──────┬───────┘                    │
│   │                          │ 分配天车/AGV                │
│   │  搬运完成上报             ↓                            │
│   │  ←────────────────┌──────────────┐                    │
│   │                   │  执行搬运      │                    │
│   │                   │  [D1→L1]      │                    │
│   │                   └──────────────┘                    │
│   │                                                        │
│   Bank层              D1出口 → 🚚 → L1入口                  │
│                                                             │
│   MES 关心：指令是否到达？搬运是否完成？耗时多久？             │
│   MES 不关心：天车走哪条轨道？怎么避让？电机转速？            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**AMHS 仿真的核心目标：**
1. **验证 MES 搬运指令下发** - 格式、时机、目标地址
2. **验证 MES 搬运完成接收** - 状态更新、异常处理
3. **测试 MES 超时/重试逻辑** - 搬运失败时的应对
4. **验证 WIP 位置追溯** - 搬运中的批次位置

#### 2.5.2 AMHS 仿真范围界定

| 层级 | 描述 | v0.1 仿真策略 |
|-----|------|--------------|
| **MES 指令层** | MES 发送"搬运 LOT001 从 Bank A 到 Bank B" | ✅ **完整仿真** |
| **AMHS 任务层** | 任务队列、分配、状态跟踪 | ✅ **简化仿真** - 黑盒延迟 |
| **物理执行层** | 天车路径规划、轨道运行、避让 | ❌ **不仿真** - 超出范围 |

#### 2.5.3 简化 AMHS 模型设计

采用**黑盒延迟模型**：

```json
// AMHS 配置 (amhs.json)
{
  "transport_profiles": [
    {
      "profile_id": "same_bay",
      "description": "同区域搬运",
      "base_duration_sec": 30,
      "variance_sec": 10,
      "failure_rate": 0.001
    },
    {
      "profile_id": "cross_bay", 
      "description": "跨区域搬运",
      "base_duration_sec": 120,
      "variance_sec": 30,
      "failure_rate": 0.002
    },
    {
      "profile_id": "interfloor",
      "description": "跨楼层搬运", 
      "base_duration_sec": 300,
      "variance_sec": 60,
      "failure_rate": 0.005
    }
  ],
  "system_capacity": {
    "concurrent_limit": 10,
    "queue_limit": 50
  },
  "failure_scenarios": [
    {
      "type": "timeout",
      "trigger_after_sec": 600,
      "probability": 0.01
    },
    {
      "type": "communication_error",
      "probability": 0.005
    }
  ]
}
```

**搬运时间计算：**
```
实际搬运时间 = base_duration + random(-variance, +variance)

示例：
- 同区域：30s ± 10s = 20~40s
- 跨区域：120s ± 30s = 90~150s  
- 跨楼层：300s ± 60s = 240~360s
```

#### 2.5.4 AMHS 状态定义

```
┌─────────────────────────────────────────────────────────────┐
│                    AMHS 搬运任务状态机                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│    ┌─────────┐   接收指令    ┌─────────┐                   │
│    │ PENDING │──────────────→│ QUEUED  │                   │
│    │ (待处理) │               │ (队列中) │                   │
│    └─────────┘               └────┬────┘                   │
│                                   │ 系统分配资源              │
│                                   ↓                          │
│                              ┌─────────┐                    │
│                              │ACTIVE   │                    │
│         ┌────────────────────│ (执行中) │                    │
│         │                    └────┬────┘                    │
│    执行失败                      │ 执行完成                   │
│         │                        ↓                          │
│    ┌────┴────┐              ┌─────────┐                    │
│    │  RETRY  │              │COMPLETE │──→ 上报 MES        │
│    │ (重试中) │              │ (已完成) │                    │
│    └────┬────┘              └─────────┘                    │
│         │                                                   │
│         └────────────────────────────────────→ 重试次数超限   │
│                                               ↓              │
│                                         ┌─────────┐         │
│                                         │  ERROR  │         │

│                                         └─────────┘         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

| 状态 | 说明 | MES 上报 |
|-----|------|---------|
| **PENDING** | 刚接收搬运指令，等待进入队列 | 可选 |
| **QUEUED** | 已进入搬运队列，等待资源 | ✅ 上报 |
| **ACTIVE** | 正在执行搬运 | ✅ 上报（可选） |
| **COMPLETE** | 搬运完成，批次已到达目标 | ✅ **必须上报** |
| **ERROR** | 搬运失败，需要人工干预 | ✅ **必须上报** |
| **RETRY** | 正在重试 | ✅ 上报 |

## 4. 数据模型

### 4.1 核心实体

#### Bank（缓存区）
```json
{
  "id": "BK_D1_IN",
  "name": "沉积站-01-入口缓存",
  "type": "INPUT",           // INPUT | OUTPUT | CENTRAL
  "associated_equipment": "D1",  // 关联设备ID，可为null
  "capacity": 3,             // 最大容纳批次数量
  "position": {"x": 280, "y": 100},
  "status": "HOLDING",       // EMPTY | HOLDING | RESERVED | TRANSFERRING
  "lots": [
    {
      "lot_id": "LOT2024001",
      "arrive_time": "2024-01-15T08:25:00Z",
      "priority": "NORMAL",
      "target_equipment": "D1"
    }
  ],
  "dispatch_policy": "FIFO"  // FIFO | PRIORITY | MES_DIRECT
}
```

**Bank 类型说明：**
| 类型 | 用途 | 典型位置 |
|-----|------|---------|
| **INPUT** | 设备入口缓存 | 设备前方，存放等待进入的批次 |
| **OUTPUT** | 设备出口缓存 | 设备后方，存放已完成等待搬走的批次 |
| **CENTRAL** | 中央缓存区 | 产线中央，作为多设备间的中转站 |

**派工策略：**
| 策略 | 说明 | 适用场景 |
|-----|------|---------|
| **FIFO** | 先进先出 | 简单流转，无优先级差异 |
| **PRIORITY** | 优先级排序 | 有紧急批次需要插队 |
| **MES_DIRECT** | 完全由 MES 决策 | 复杂调度逻辑 |

## 5. 仿真引擎设计

### 5.1 时间推进机制

```
┌────────────────────────────────────────────┐
│              仿真时间管理器                  │
├────────────────────────────────────────────┤
│  模式: [暂停] [1x] [10x] [100x] [Max]      │
│                                            │
│  虚拟时间: 2024-01-15 10:30:00              │
│  实际经过: 00:05:32                         │
│  仿真速度: 100x                             │
└────────────────────────────────────────────┘
```

- **暂停**：仿真停止，但可单步执行
- **1x**：虚拟时间 = 实际时间
- **10x/100x**：加速运行，适合验证长流程
- **Max**：最大速度，跳过渲染，最快速度完成

### 5.2 设备状态机

```
                    ┌─────────────┐
         ┌─────────→│   IDLE      │←─────────┐
         │  工单完成  │  (空闲)      │          │
         │          └──────┬──────┘          │
         │                 │ 收到工单         │
         │                 ↓                 │
         │          ┌─────────────┐          │
         │    ┌────→│   SETUP     │          │
         │    │     │  (准备中)    │          │
         │    │     └──────┬──────┘          │
         │    │            │ 准备完成        │
         │    │            ↓                 │
         │    │     ┌─────────────┐          │
         │    │     │   RUNNING   │          │
         │    │     │  (运行中)    │          │
         │    │     └──────┬──────┘          │
         │    │            │ 工艺完成        │
         │    │            ↓                 │
         │    └─────┌─────────────┐          │
         │          │  COMPLETE   │          │
         └──────────│  (完成)      │          │
                    └─────────────┘          │
                           │                 │
          ┌────────────────┼────────────────┘
          ↓                ↓
    ┌─────────────┐  ┌─────────────┐
    │    ERROR    │  │ MAINTENANCE │
    │   (异常)     │  │   (维护中)   │
    └──────┬──────┘  └─────────────┘
           │ 人工处理
           ↓
      [回到IDLE]
```

### 5.3 事件系统

仿真引擎基于离散事件驱动：

| 事件类型 | 触发条件 | 动作 |
|---------|---------|------|
| LOT_ARRIVE | 新批次到达入口 | 向 MES 上报，请求派工 |
| **BANK_LOT_ARRIVE** | **批次到达 Bank** | **上报 MES，触发派工决策** |
| **BANK_DISPATCH** | **MES 下发派工指令** | **锁定 Bank 批次，准备流转** |
| **BANK_LOT_DEPART** | **批次离开 Bank** | **更新 Bank 状态，上报 MES** |
| EQUIPMENT_START | 设备开始工艺 | 状态→RUNNING，上报MES |
| EQUIPMENT_COMPLETE | 工艺完成 | 状态→COMPLETE，批次移出至输出Bank |
| LOT_MOVE | 物料需要流转 | 检查目标 Bank 容量，执行移动 |
| MES_DISPATCH | 收到MES派工指令 | 将Lot加入设备队列或Bank |
| **EAP_EVENT** | **设备状态变化** | **按SECS格式上报S6F11事件** |
| **EAP_COMMAND** | **收到远程指令** | **执行S2F41指令并响应** |

**Bank 相关的典型事件流：**
```
[设备A COMPLETE] 
      ↓
[LOT移出至 Bank-A-OUT] 
      ↓
[BANK_LOT_ARRIVE 事件] → 上报 MES
      ↓
[MES 决策: 目标设备B]
      ↓
[BANK_DISPATCH_CONFIRM 事件]
      ↓
[检查 Bank-B-IN 容量]
      ↓
[LOT_MOVE: Bank-A-OUT → Bank-B-IN]
      ↓
[BANK_LOT_DEPART + BANK_LOT_ARRIVE 上报]
      ↓
[设备B从 Bank-B-IN 取货 → SETUP → RUNNING]
```

## 附录：术语表

| 术语 | 解释 |
|-----|------|
| MES | Manufacturing Execution System，制造执行系统 |
| WIP | Work In Process，在制品 |
| OEE | Overall Equipment Effectiveness，设备综合效率 |
| Lot | 批次，一组晶圆 |
| Recipe | 工艺配方 |
| FSM | Finite State Machine，有限状态机 |
| **Bank** | **缓存区/暂存区，WIP流转的缓冲节点** |
| **FIFO** | **First In First Out，先进先出调度策略** |
| **Load Port** | **设备上下料端口，对应仿真中的 Bank** |
| **Dispatch** | **派工，MES 决定将批次发往何处** |
| **AMHS** | **Automated Material Handling System，全自动物料搬运系统** |
| **Transport** | **搬运任务，AMHS 执行的批次移动操作** |
| **Area** | **区域，工厂的逻辑分区（如光刻区、刻蚀区）** |
| **Work Area** | **工作区域，更细粒度的逻辑分组** |
| **EAP** | **Equipment Automation Program，设备自动化程序** |
| **SECS** | **SEMI Equipment Communications Standard，半导体设备通信标准** |
| **GEM** | **Generic Equipment Model，通用设备模型** |
| **CEID** | **Collection Event ID，收集事件标识** |
| **SVID** | **Status Variable ID，状态变量标识** |
| **PPID** | **Process Program ID，工艺程序(配方)标识** |
| **SML** | **SECS Message Language，SECS 消息语言** |
