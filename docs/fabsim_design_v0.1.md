## 1. 项目概述

### 1.1 目标
FabSim是一款半导体制造工厂的离散事件仿真系统，用于验证产线调度策略、预测产能瓶颈、优化生产排程。

### 1.2 核心定位
- **不是**：3D可视化、MES系统、真实设备控制
- **是**：2D拓扑仿真、Lot流转模拟、调度算法验证

### 1.3 MVP边界

| 做（In Scope） | 不做（Out of Scope） |
|----------------|---------------------|
| 单条产线仿真 | 多条产线联动 |
| 6种基础设备 | 全部14+种设备 |
| FIFO调度 | 复杂智能调度 |
| 单层工艺 | 多层循环（后续版本） |
| 本地JSON配置 | MES实时对接 |
| 单机运行 | 联网协作 |

## 3. 数据模型

### 3.1 产线布局 (Layout)

```yaml
产线布局:
  machines: [设备列表]
    - id: 唯一标识 (如 "M1")
      type: 设备类型 (clean/deposition/lithography/etching/heat/inspect)
      name: 显示名称 (如 "清洗机")
      x, y: 屏幕坐标位置
      process_time: 加工时间 (分钟)
  
  buffers: [缓冲列表]
    - id: 唯一标识 (如 "B1")
      name: 显示名称
      x, y: 屏幕坐标位置
      capacity: 最大容量 (整数)
  
  connections: [连接关系]
    - from: 起点ID
      to: 终点ID
```

设备类型:
  - id: clean

  - id: deposition

### 3.4 仿真实体

#### Lot（批次）
```yaml
Lot:
  id: 唯一编号 (如 "L0001")
  enter_time: 进入系统时间戳
  current_step: 当前执行到第几步 (0-6)
  status: 状态枚举
    - waiting: 等待加工
    - processing: 加工中
    - completed: 已完成
  location: 当前位置 (设备ID或缓冲ID)
```

#### 设备状态
```yaml
设备状态:
  id: 设备标识
  status: 状态枚举
    - idle: 空闲
    - processing: 加工中
  current_lot: 当前加工的Lot (null表示空闲)
  queue: 等待队列 (Lot列表)
  busy_until: 加工完成时间戳
  total_processed: 累计加工数量
```

#### 缓冲状态
```yaml
缓冲状态:
  id: 缓冲标识
  queue: 当前存放的Lot列表
  capacity: 最大容量
```

## 4. 核心流程

### 4.1 仿真主循环

```
每帧执行：
  1. 计算仿真时间增量 (实际增量 × 时间加速倍率)
  2. 推进仿真时间
  3. 检查设备完成事件
     - 对每台加工中的设备：
       - 如果当前时间 ≥ busy_until：
         - 标记设备为空闲
         - Lot完成当前步骤
         - 如果还有下一步：Lot移入下一缓冲
         - 如果无下一步：标记Lot完成
  4. 从缓冲移动Lot到设备队列
     - 对每个缓冲：
       - 将缓冲中的Lot移入下一台设备的等待队列
  5. 分配空闲设备
     - 对每台空闲且有待处理队列的设备：
       - 取出队列第一个Lot
       - 标记设备为加工中
       - 计算busy_until = 当前时间 + process_time
  6. 更新统计信息 (WIP、吞吐量等)
  7. 触发UI更新
```

### 4.2 Lot生命周期

```
生成 → 进入首设备队列 → 等待 → 加工中 → 步骤完成
                                        ↓
                              是否有下一步？
                           是 → 进入缓冲 → 移入下一设备队列
                           否 → 标记完成
```

### 4.3 调度策略 (FIFO)

```
规则：
1. 设备选择Lot：先到先得，队列中最早等待的Lot优先
2. Lot选择设备：按工艺流程固定路径，无选择逻辑
3. 缓冲到设备：缓冲中Lot按到达顺序移入设备队列
```

## 6. 配置说明

### 6.1 修改产线布局

编辑产线布局JSON文件：
- 调整`x, y`改变设备位置
- 调整`process_time`改变加工时间
- 增删`machines`列表改变设备数量
- 增删`buffers`列表改变缓冲数量
- 修改`connections`改变流转路径

### 6.2 修改工艺配方

编辑工艺配方JSON文件：
- 调整`steps`中的`time`改变各步骤时间
- 增删`steps`改变工艺流程
- `machine_type`必须对应设备类型ID

### 6.3 调整初始Lot数量

在仿真引擎初始化时设置：
- MVP默认生成5个Lot
- 可调整初始数量验证不同负载

## 7. 统计指标

| 指标 | 计算方法 |
|------|----------|
| **仿真时间** | 累计推进的仿真分钟数 |
| **WIP** | 状态不为"已完成"的Lot数量 |
| **吞吐量** | 已完成Lot数 ÷ 仿真时间(小时) |
| **平均CT** | 已完成Lot的(完成时间-进入时间)平均值 |
| **设备利用率** | 设备处于"加工中"状态的时间占比 |
