# 半导体工厂仿真系统计划书

> 项目名称：FabSim（Fab Simulation System）
> 技术栈：Godot 4.x + GDScript / C#

---

## 一、项目定位

**FabSim** 是一款面向半导体制造工厂的数字化仿真平台，核心目标：
- 🏭 **可视化**：3D 还原晶圆厂产线布局、设备运行状态
- 🔗 **可联动**：与真实 MES 系统双向交互，实现虚实同步
- 🧪 **可验证**：在虚拟环境中测试 MES 策略、排程算法、异常处理流程

---

## 二、核心功能模块

### 2.1 工厂建模层（Factory Modeling）

| 功能 | 说明 |
|------|------|
| **产线编辑器** | 拖拽式搭建晶圆产线，支持添加设备、缓冲区、传输路径 |
| **设备模板库** | 预设光刻机、刻蚀机、薄膜沉积、清洗设备等标准模型 |
| **洁净室环境** | 模拟温湿度、洁净度等级、气流方向等环境因素 |
| **3D 可视化** | 实时渲染设备运行、晶圆流动、AGV 运输等场景 |

### 2.2 工艺仿真层（Process Simulation）

| 功能 | 说明 |
|------|------|
| **工艺流定义** | 配置工艺流程（Recipe）、步骤参数、设备约束 |
| **晶圆追溯** | 每片晶圆的 WIP 状态、工艺历史、位置追踪 |
| **时间仿真** | 支持加速/减速/暂停，模拟小时级/天级生产周期 |
| **异常注入** | 人工触发设备故障、工艺异常、物料短缺等场景 |

### 2.3 调度仿真层（Scheduling Simulation）

| 功能 | 说明 |
|------|------|
| **排程算法** | 内置 FIFO、SPT、CR、OAS 等经典规则，支持自定义算法 |
| **资源分配** | 设备负载均衡、维护窗口规划、批次合并优化 |
| **实时看板** | WIP 监控、设备 OEE、Cycle Time 等 KPI 实时展示 |
| **策略 A/B 测试** | 对比不同调度策略下的产线效率差异 |

### 2.4 数据接口层（Integration Layer）

| 功能 | 说明 |
|------|------|
| **MES 连接器** | 通过 REST/GraphQL/SECS-GEM 与真实 MES 对接 |
| **数据同步** | 双向同步工单、工艺参数、设备状态、生产实绩 |
| **影子模式** | 仿真系统跟随真实产线运行，验证策略有效性 |
| **沙盒模式** | 完全脱离真实系统，独立运行仿真实验 |

### 2.5 分析报表层（Analytics & Reporting）

| 功能 | 说明 |
|------|------|
| **生产报表** | Throughput、WIP 曲线、设备利用率、Cycle Time 分布 |
| **瓶颈分析** | 自动识别产线瓶颈工位，输出优化建议 |
| **对比分析** | 历史数据 vs 仿真预测、真实产线 vs 仿真产线 |
| **导出能力** | 支持 Excel、CSV、PDF 报表导出 |

### 3.2 数据交互接口定义

#### 3.2.1 MES → FabSim（下行数据）

| 数据类型 | 内容 | 频率 |
|----------|------|------|
| **工单信息** | Lot ID、产品型号、工艺路线、优先级 | 实时推送 |
| **工艺参数** | Recipe 参数、工艺时间、质量规格 | 按需拉取 |
| **设备状态** | 设备 ID、当前状态、维护计划 | 实时推送 |
| **物料信息** | 物料批次、库存位置、有效期 | 定时同步 |
| **人员信息** | 操作员资质、班组排班 | 定时同步 |

#### 3.2.2 FabSim → MES（上行数据）

| 数据类型 | 内容 | 用途 |
|----------|------|------|
| **仿真预测** | 预计完成时间、瓶颈预警 | 辅助排程决策 |
| **策略建议** | 优化后的调度方案 | 策略推荐 |
| **异常模拟** | 故障场景推演结果 | 应急预案验证 |
| **效能评估** | 新策略的 ROI 预测 | 投资决策支持 |

### 3.3 联动模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **影子模式（Shadow）** | 仿真系统实时跟踪真实产线，预测未来状态 | 实时预警、策略预演 |
| **回放模式（Playback）** | 导入历史数据，复盘特定生产事件 | 故障复盘、培训演示 |
| **沙盒模式（Sandbox）** | 完全离线运行，自由修改参数 | 新策略测试、"What-if" 分析 |
| **混合模式（Hybrid）** | 部分真实 + 部分仿真 | 新设备验证、扩产规划 |

## 四、仿真实现机制

### 4.1 核心仿真引擎架构

```
┌────────────────────────────────────────────────────────────┐
│                     仿真引擎核心层                          │
├────────────────────────────────────────────────────────────┤
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────┐ │
│  │   离散事件调度器   │  │   实体对象管理器   │  │  时间控制器   │ │
│  │  (Event Queue)   │  │  (Entity Pool)   │  │ (Time Warp) │ │
│  └────────┬────────┘  └────────┬────────┘  └──────┬──────┘ │
│           │                    │                  │        │
│           └────────────────────┼──────────────────┘        │
│                                │                           │
│                    ┌───────────▼───────────┐               │
│                    │     仿真核心循环       │               │
│                    │  1. 获取下一事件       │               │
│                    │  2. 推进仿真时间       │               │
│                    │  3. 处理事件逻辑       │               │
│                    │  4. 更新实体状态       │               │
│                    │  5. 触发后续事件       │               │
│                    └───────────────────────┘               │
└────────────────────────────────────────────────────────────┘
```

### 4.2 关键技术实现

#### 4.2.1 离散事件仿真（DES）

```gdscript
# 伪代码示例：Godot 中的离散事件处理
class_name EventScheduler

var event_queue: PriorityQueue  # 按时间排序的事件队列
var current_time: float = 0.0   # 仿真当前时间
var time_scale: float = 1.0     # 时间加速倍率

func run_simulation():
    while not event_queue.is_empty():
        var next_event = event_queue.pop()
        current_time = next_event.time
        process_event(next_event)
        
func process_event(event: SimEvent):
    match event.type:
        "LOT_ARRIVE":
            handle_lot_arrival(event.data)
        "MACHINE_COMPLETE":
            handle_machine_complete(event.data)
        "TRANSPORT_DONE":
            handle_transport_done(event.data)
```

#### 4.2.2 实体建模

| 实体类型 | 属性示例 | 行为 |
|----------|----------|------|
| **Lot（晶圆批次）** | ID、数量、当前工步、优先级、状态 | 排队、加工、搬运 |
| **Machine（设备）** | ID、类型、状态、能力、维护窗口 | 加工、故障、维护 |
| **Buffer（缓冲区）** | ID、容量、当前数量、类型 | 存储、FIFO/LIFO |
| **Transport（传输）** | ID、速度、容量、当前任务 | 搬运、AGV 调度 |
| **Operator（操作员）** | ID、技能、班次、当前任务 | 人工操作、检验 |

#### 4.2.3 时间控制机制

| 功能 | 实现方式 |
|------|----------|
| **加速仿真** | time_scale = 10x/100x/1000x，跳过空闲等待 |
| **精确时刻** | 事件发生时刻精确到秒级/分钟级 |
| **快照保存** | 任意时刻保存仿真状态，支持回溯 |
| **并行仿真** | 多场景并行运行，对比结果 |

### 4.3 可视化渲染策略

| 层级 | 实现方式 | 细节程度 |
|------|----------|----------|
| **鸟瞰层** | 2D 拓扑图 + 热力图 | 产线整体流量、设备负载 |
| **产线层** | 3D 设备模型 + 动画 | 设备运行、晶圆流动、AGV |
| **设备层** | 详细 3D 模型 + 状态面板 | 内部结构、实时参数 |
| **数据层** | HUD 覆盖 + 图表 | KPI、告警、趋势 |

## 七、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| **MES 协议不统一** | 高 | 设计灵活的适配层，支持多种协议 |
| **仿真精度 vs 性能** | 中 | 分层仿真，关键工位高精度、非关键工位简化 |
| **3D 资源制作成本** | 中 | 复用模板 + 程序化生成 |
| **Godot Web 版性能** | 中 | 复杂场景用桌面版，Web 版做轻量展示 |
